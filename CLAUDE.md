# hyperframes-integration — Operating Manual

This file is the project-level contract. It assumes you have James's global
`~/.claude/CLAUDE.md` loaded (Learning Mode, Ironclad Workflow, ASTGL flagging).
Where this file is more specific, this file wins. Read the per-composition
`CLAUDE.md` inside each `compositions/<id>/` too — those are vendor-generated by
`hyperframes init` and carry the framework's own rules; do not rewrite them,
they get regenerated.

## What this project is

A **pilot** to wire HeyGen's HyperFrames (HTML+GSAP → deterministic MP4) into
James's existing content pipelines as a *video media stage* — not a new
standalone pipeline. Locked Tier 1 order (no skipping):

1. ✅ `compositions/pki-lifecycle/` — hand-authored warm-start (2026-05-31)
2. ⏳ `astgl-publish` video-generator stage — 9 vertical promos (3 latest ASTGL
   posts × 3 hook variants A="the problem" / B="the insight" / C="the tool/result")
3. ⏳ `substack-scheduler` release-walkthrough videos on tagged release

Full plan context: auto-memory `pilot-plan.md`. Every friction point here is
potential ASTGL article material — flag `[ASTGL CONTENT]` liberally.

**Git:** initialized 2026-07-08, branch `main`, no remote yet. The Ironclad
Workflow applies: never commit directly to main once a remote with branch
protection exists — until then, still branch for anything non-trivial.
Ignored by design: rendered MP4s (reproduce via `npm run render`) and vendor
skills in `.agents/skills/` (reinstall via `npx skills add
heygen-com/hyperframes`; `skills-lock.json` is tracked and pins them).
Creating the GitHub remote (`gh repo create`) and pushing is James's call —
the ASTGL article links to `github.com/jmcruce/hyperframes-integration`.

## Conventions (James's, plus additions)

### Established by James — follow exactly

- **Version pin.** Every command goes through the composition's `package.json`
  scripts, which pin `npx --yes hyperframes@0.6.64`. HyperFrames is pre-1.0 and
  breaks between versions. Never run bare `npx hyperframes` (floats to latest).
  Version bumps are a decision James makes, not you.
- **Skills first.** Before writing or editing any composition, invoke the
  relevant vendor skill (`/hyperframes`, `/gsap`, `/hyperframes-media`,
  `/hyperframes-cli`). They encode framework semantics that are not in generic
  web knowledge. Writing composition HTML from memory produces broken output.
- **Layout before animation.** Build the static end-state first: every element
  positioned at its most-visible frame, no GSAP. Render/preview that. Only then
  add tweens. Animating a layout you haven't seen static means overlaps you
  discover after burning a render.
- **`npm run check` gates everything.** lint → validate → inspect, zero errors,
  after every composition edit and before every render. The three real bug
  classes (transform conflicts, track collisions, font fallbacks) are all
  lint-catchable; a skipped check is a wasted render cycle.
- **Local-first media.** TTS is local Kokoro (`am_michael`), transcription is
  local Whisper, rendering is local (M3 Ultra, ~1.5× real-time landscape,
  ~1.7× portrait, 6 workers). No cloud render, no cloud TTS, no API spend
  without asking.
- **Dual deliverables.** Every finished video ships 1920×1080 (blog embed) and
  1080×1920 (Substack Notes / mobile). Naming:
  `<id>-<W>x<H>[-narrated][-vN].mp4`. New cut = bump `-vN`; never overwrite or
  delete an existing MP4.
- **Narration voice rules** (from James's own corrections — non-negotiable):
  - ≥0.5s of silence before the first word (`ffmpeg adelay=500|500`) — Kokoro
    clips the start otherwise.
  - Air + tail padding at the end — the last word gets clipped otherwise.
  - The sign-off is a standalone sentence, never grammatically attached to the
    preceding line ("Another walkthrough from As The Geek Learns." — not
    "...begins again. As The Geek Learns.").
  - Never write "astgl" where TTS will pronounce it — it says "asccul." Spell
    it `A-S-T-G-L` or avoid the URL in speech entirely.
- **Audio-first timing.** Generate and pad the narration, `ffprobe` its real
  duration, then set composition duration and pace timeline cues to the audio —
  never the other way around.

### Added by Claude — treat as equally binding

- **One track table per composition.** Before writing timed elements, write a
  comment block mapping track index → element (e.g. 1=title, 2=subtitle,
  3=diagram, 4=outro, 5=narration audio). Audio always gets its own track.
  Tracks are *scheduling lanes*, not visual layers.
- **Portrait parity check.** `index.html` and `index-portrait.html` are
  hand-synced siblings. Any content/timing change to one is not done until it's
  in both (portrait usually needs a `transform: scale(...)` on the diagram
  container, not a re-layout).
- **cd before CLI.** Always run HyperFrames commands from inside the
  composition directory. The CLI silently treats a missing input file path as
  literal text — `tts narration.txt` from the wrong cwd produces 1.6s of audio
  *saying "narration dot t x t"*. After any TTS run, `ffprobe` the output and
  sanity-check duration against word count (~2.5 words/sec).
- **TTS preflight.** `hyperframes tts` needs `kokoro-onnx` and `soundfile` in
  the active Python (`pip3 install kokoro-onnx soundfile`). Check before a cold
  run instead of failing mid-pipeline.
- **Preview server is long-running.** `npm run dev` blocks forever — always
  `run_in_background: true`. Foreground it and it times out, killing the preview.
- **Render cost awareness.** A render is minutes of compute; treat it like a
  deploy, not a compile. Everything cheap (check, snapshot, preview) runs
  before anything expensive (render), and you never render twice to discover
  what one snapshot would have shown.

## Mistakes a weaker model WILL make here — named, with the preventing rule

1. **The transform-conflict bug.** Positioning an element with
   `transform: translate(-50%, ...)` then animating it with GSAP — GSAP
   overwrites the *entire* transform and silently destroys the centering.
   → **Rule:** position with `left`/`top` + negative margins; `transform`
   belongs to GSAP alone (the one sanctioned exception: a static
   `scale()` on a non-animated container, as in `.diagram`).
2. **The track-collision bug.** Putting two concurrently-visible clips on the
   same `data-track-index` because "they're on different layers." Tracks are
   scheduling lanes; same-track overlap = flicker.
   → **Rule:** every concurrently-visible clip gets its own track index, per
   the track table. z-order is CSS `z-index`, unrelated.
3. **The font-fallback bug.** Writing web-idiomatic stacks
   (`-apple-system, BlinkMacSystemFont, ...`). Headless Chrome has no system
   fonts; the render silently falls back to generic sans.
   → **Rule:** one embeddable font per family role (e.g. `"Inter", sans-serif`),
   or explicit `.woff2` files. Nothing the compiler can't resolve.
4. **The missing-clip-class bug.** Adding `data-start`/`data-duration` and
   forgetting `class="clip"` — the framework controls visibility through that
   class; without it the element is visible for the whole video.
   → **Rule:** timed ⇒ `class="clip"` + all three of `data-start`,
   `data-duration`, `data-track-index`. No exceptions.
5. **The unregistered-timeline bug.** Creating a GSAP timeline that autoplays,
   or forgetting `window.__timelines["id"] = tl`.
   → **Rule:** every timeline is `gsap.timeline({ paused: true })` and
   registered on `window.__timelines`. The framework drives playback by seeking.
6. **Nondeterminism.** `Date.now()`, `Math.random()`, network fetches, unseeded
   anything. The whole value proposition is identical frames on every render.
   → **Rule:** deterministic logic only. Same input → same MP4 hash.
7. **The aspect-ratio assumption.** Believing `--resolution` can produce the
   portrait cut. It only *scales* — aspect must match the source.
   → **Rule:** portrait is its own file (`index-portrait.html`); rendering it
   means the swap dance (or `/hf-render-pack`, which does it safely).
8. **Rendering to verify.** Using a full render to check layout or text overlap.
   The horizontal title-overwrite bug shipped twice this way.
   → **Rule:** `npm run check` + `hyperframes snapshot` of key frames first;
   render only when both are clean. Then still *watch* the MP4 (or snapshot it)
   before calling it done.
9. **Editing one aspect, shipping both.** Fixing the landscape file and
   rendering portrait from the stale sibling.
   → **Rule:** portrait parity check above — the diff between the two files
   should be exactly viewport, scale factor, and layout positions.
10. **Re-cueing amnesia.** Regenerating narration and keeping old timeline cues.
    The 500ms pad shifts *every* cue; a new script shifts all of them.
    → **Rule:** narration change ⇒ re-measure with `ffprobe` ⇒ update `<audio>`
    `data-duration`, root `data-duration`, and re-pace cues. This is what
    `/hf-narrate` automates.
11. **Ironclad reflexes in a non-repo.** Trying to create branches or PRs here.
    → **Rule:** no git ops until the repo is initialized (ask first).
12. **Version drift.** "Helpfully" upgrading hyperframes or un-pinning `npx`.
    → **Rule:** 0.6.64 until James says otherwise.
13. **AI-slop design defaults.** Gradient text, cyan-on-dark, neon accents,
    identical card grids, pure `#000`/`#fff`, everything centered.
    → **Rule:** follow `.agents/skills/hyperframes/house-style.md` — declare
    bg/fg/accent up front, one accent hue, tint neutrals toward it, 2–5 ambient
    decoratives per scene with slow motion. If reaching for a lazy default,
    justify it for *this* content or pick again.

## Quality bar — checkable, per deliverable

A deliverable is DONE only when every box below is verifiably true. "Looks
right" is not a criterion; each item is a command you ran or a fact you observed.

**Composition (`index.html` / `index-portrait.html`)**
- [ ] `npm run check` exits clean: lint 0 errors, validate WCAG AA pass on all
      text, inspect 0 overflow findings (warnings reviewed, each one either
      fixed or explained in your summary)
- [ ] Every timed element has `class="clip"` + `data-start` + `data-duration`
      + `data-track-index` (grep for `data-start` and verify)
- [ ] Track table comment present and matches actual `data-track-index` usage
- [ ] Timeline is `paused: true` and registered on `window.__timelines`
- [ ] No `Date.now`, `Math.random`, `fetch(`, external requests beyond the
      pinned GSAP CDN (grep for each)
- [ ] `transform` appears in CSS only on non-animated containers
- [ ] Font stack contains only embeddable faces
- [ ] Portrait sibling updated in the same session as any landscape change

**Narration (`narration.txt` + `narration.wav`)**
- [ ] Script contains no TTS traps: "astgl" spelled `A-S-T-G-L` or absent;
      numbers/acronyms written the way they should be *spoken*
- [ ] Sign-off is a standalone final sentence
- [ ] WAV starts with ≥0.5s silence and ends with ≥0.5s tail (verify with
      ffprobe/waveform, not by assumption)
- [ ] `ffprobe` duration recorded, and `<audio data-duration>` matches it
- [ ] Root `data-duration` ≥ audio duration + ~1.5s visual hold
- [ ] You (or James) actually listened to the ends — first word and last word
      intact. If you can't listen, say so explicitly and ask James to.

**Render (MP4s)**
- [ ] Rendered only after the composition checklist passed
- [ ] Both aspect ratios produced, named `<id>-<W>x<H>[-narrated][-vN].mp4`
- [ ] No existing MP4 overwritten — new version suffix instead
- [ ] Key frames snapshotted or the file watched: title readable, no text
      overlap, outro intact, audio in sync at start/middle/end
- [ ] Wall-clock render stats reported (James tracks ~1.5×/1.7× real-time
      baselines; a big deviation is a signal, not noise)

**Article draft (`articles/`)**
- [ ] Frontmatter: title, subtitle, publication: ASTGL, `draft: true`, date, tags
- [ ] Written through the `astgl-writer` voice skill, not freehand
- [ ] Concrete numbers kept concrete (render times, lint counts, versions) —
      the specifics are the ASTGL brand
- [ ] Never published/synced anywhere without explicit go-ahead

## When uncertain — exact escalation rules

Aesthetic and voice judgments are **unknown-knowns for James: he knows it when
he sees/hears it.** Don't ask abstract questions; produce something reactable.

1. **Design/motion/palette choice with ≥2 defensible options** → build the
   cheapest reactable artifact (snapshot PNGs, a 5s preview, 2–3 static
   variants), present them, let James pick. Never pick a subjective winner
   silently on a deliverable.
2. **Voice/pacing** → default is `am_michael`, speed 1.0. Any change: generate
   short samples and give James the file paths to play. Never describe audio
   in prose and call it reviewed.
3. **`npm run check` won't come clean** after a genuine fix attempt → STOP.
   Report the exact findings and your analysis. Never render over a dirty
   check, never suppress or talk around a lint error.
4. **Anything that publishes or leaves the machine** (`npm run publish`,
   `hyperframes cloud`, Substack sync, `git push` someday) → explicit
   confirmation every time. No standing authorization exists.
5. **Deleting or overwriting any render, narration.wav, or article** → don't.
   Version alongside. If cleanup is genuinely needed, list the files and ask.
6. **Framework behavior you're not sure about** → in order: the loaded skill →
   `npx hyperframes docs <topic>` → https://hyperframes.heygen.com/llms.txt
   (discover URLs from the index, don't guess). If still unresolved, say
   "the docs don't cover this" and propose a small experiment — don't invent
   framework semantics.
7. **First-time cost** (model downloads, >5min renders, new pip/npm deps) →
   name the cost before incurring it, once; after James approves that class of
   cost, proceed without re-asking in the session.
8. **Scope forks** (e.g. "should this become a Tier-2 item?", new repo, new
   pipeline stage) → the pilot plan order is locked; surface the idea, don't
   act on it. James decides sequencing.
9. **Genuinely blocked >2 attempts** on the same error → stop the loop, write
   up what you tried and what you observed, ask. A wrong guess in a rendered
   video costs a full re-render plus James's review time.
